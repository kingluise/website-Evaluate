3. HTML and CSS Validation

The HTML and CSS code was validated using the W3C validators to ensure compliance with web standards.

3.1. HTML Validation

The HTML code was checked using the W3C Nu HTML Checker.

3.1.1. Initial Check of Homepage and Attraction Page

When the homepage and attraction page HTML were first checked, the tool gave some "Info" messages. These aren't serious errors that break the page, but they point to areas where the code could be written in a better way.

Specifically, the tool mentioned two things:

  • Extra slashes in tags: Some tags, like those for images (<img>), had an extra slash at the end (e.g., <img ... />). It's like writing "okay/" instead of "okay" - it's not wrong, but it's not the usual way to do it in HTML5.
  • Missing quotes around information: When we give instructions to HTML tags, like specifying the character set (<meta charset=UTF-8>), the values should be in quotes. It's like saying "Please use the font 'Arial'" instead of "Please use the font Arial". Browsers usually understand it either way, but quotes make the code cleaner and more reliable.

These "Info" messages are suggestions for making the HTML code cleaner and more consistent, so it works well in all browsers.

3.1.2. After Corrections

After addressing the "Info" messages by removing the trailing slashes from void elements and ensuring all attribute values were enclosed in quotation marks, the HTML for the homepage and/or attraction page was re-validated. The W3C Nu HTML Checker now reports: "Document checking completed. No errors or warnings to show." This indicates that the HTML code now adheres to the recommended HTML5 standards.

3.1.3. Validation Results - Accommodation, Tips, and Contact Pages

The HTML code for the accommodation, tips, and contact pages was also checked using the W3C Nu HTML Checker. These pages displayed no initial errors or warnings. The validator reported: "Document checking completed. No errors or warnings to show." This indicates that the HTML code for these pages was already compliant with recommended HTML5 standards.

4. Readability

The readability of the website content was assessed to ensure it is clear and accessible to a wide audience.

The following measures were taken to enhance readability:

  • Font Choice and Size: The main body text uses Arial, a sans-serif font, at a size of 16px. Sans-serif fonts are generally considered easier to read on screen. The 16px font size provides sufficient readability without being too small.
  • Color Contrast: Sufficient color contrast is used between text and background. For example, dark gray text is used on a white background to ensure that the text is easily distinguishable.
  • Text Structure: The content is structured using headings (H1, H2, H3) and paragraphs (P) to create a clear hierarchy and break up large blocks of text. This makes it easier for users to scan and understand the information.
  • Line Length: Paragraphs are kept to a reasonable line length to prevent eye strain. Long lines can be difficult to read, so keeping them relatively short improves readability.
  • Language Clarity: The language used throughout the website is clear, concise, and avoids jargon. This ensures that the content is accessible to a general audience.

The "Discover Dubai" website has been evaluated for browser compatibility, screen size/device view testing, HTML and CSS validation, and readability. The website demonstrates good compatibility across different browsers and responsiveness across various screen sizes, tested using Chrome DevTools. The HTML and CSS code adheres to web standards after corrections. The content is designed to be readable and accessible. Overall, the evaluation indicates that the website meets the requirements for usability and accessibility.
